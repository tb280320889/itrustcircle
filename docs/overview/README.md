# 01 概览

## 1. 项目定位

iTrustCircle 是一个**被动触发的紧急预警系统**，解决现有 SOS 方案在"突发失能 / 被抢手机 / 无法操作"场景下的覆盖不足。

### 核心创新
- 基于 **BLE 断联 + 倒计时无人取消** 的被动触发机制
- 三角色协作模型：Tower（塔台）+ Sentinel（哨兵）+ Contact（联系人）
- 可靠投递：离线队列 + 重试 + 恢复后补发

## 2. 项目现状

- 仓库结构：monorepo（当前仅 apps/mobile）
- 技术栈：SvelteKit + Capacitor v7（Android MVP）
- 目标阶段：Hello World 已跑通，进入 MVP 文档规范与蓝图沉淀

## 3. MVP 边界

- 目标：稳定、低心智、低功耗地把"断联 → 倒计时 → 上报 → 通知信任人"跑通
- 非目标：不做跨层架构重构；不引入复杂权限体系；本轮专注文档层

## 4. 三角色模型

### Tower（塔台）
- **设备**：家里旧手机/平板
- **职责**：接收 AlertEvent、SQLite 存档、触发通知（MVP：Email）
- **关键能力**：POST /api/alerts 接收、状态页、诊断导出

### Sentinel（哨兵）
- **设备**：随身手机（未来扩展 WearOS）
- **职责**：监测 BLE 连接、断联触发倒计时、上报报警事件
- **关键能力**：后台守护、离线队列、权限自检

### Contact（联系人）
- **设备**：无需独立 App
- **职责**：接收通知并按指引行动
- **关键能力**：通过 Email/IM/SMS 获取关键信息（谁/何时/何地/为何）
